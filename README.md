# SACRD Food Access Disparities Analysis

This project analyzes **food-related program access and demand in San Antonio**, using web analytics from the [San Antonio Community Resource Directory (SACRD)](https://sacrd.org) alongside a directory of local food programs. The core output is the **Gap Score**, a metric that identifies ZIP codes where demand for food-related programs is high relative to the food-related services available there.

The analysis includes data cleaning/aggregation scripts, exploratory clustering notebooks, and an interactive Streamlit dashboard for exploring program access, demand, and community need by ZIP code.

This project was built during a data science internship with **H-E-B's Community Innovation Scholars Program**, in partnership with SACRD.

---

## A note on the data

The real interaction and program data used in the original analysis was provided by SACRD under a data-use agreement and **is not included in this repository** — it isn't mine to redistribute.

In its place, this repo ships a **synthetic dataset with the same schema** (`Data/sample_data/`, generated by `Data/sample_data/generate_sample_data.py`): fictional organizations, addresses, and interaction counts across a handful of real, public San Antonio ZIP codes. All the cleaning, gap-score, and dashboard code is unmodified project logic — it just runs against fake data by default, so anyone can clone this repo and see the full pipeline work end-to-end without needing access to the real dataset.

## The Gap Score

For each ZIP code and month, the pipeline computes:

1. **Raw demand** — interaction counts plus a weighted count of "shares" (`share_weight`, default 10x), aggregated from site interactions.
2. **Demand score** — raw demand normalized per 1,000 residents, using ZIP-level population.
3. **Gap Score** — demand score divided by `1 + log(1 + program_count)`, so ZIP codes with high demand *and* few nearby food programs score highest.

In short: **a high Gap Score marks a ZIP code where a lot of people are looking for food help, but comparatively few food programs are actually there to meet that need.**

## Running the Dashboard

```bash
pip install -r requirements.txt
```

By default, everything (the dashboard and the underlying cleaning/gap-score pipeline) reads from the bundled synthetic dataset at `Data/sample_data/` — no setup required.

```bash
streamlit run Dashboard_Final/full_dashboard.py
```

This also works run from inside `Dashboard_Final/` (`cd Dashboard_Final && streamlit run full_dashboard.py`), and either way from a Streamlit Community Cloud deployment — set the app's main file path to `Dashboard_Final/full_dashboard.py`.

If you have your own dataset in the same schema, point the pipeline at it instead of the sample data:

```bash
export SACRD_DATA_DIR=/path/to/your/Data   # Windows: set SACRD_DATA_DIR=...
```

## Real-World Outcome

Using the real SACRD dataset, this analysis identified **5 ZIP codes with statistically significant upward trends** in food-program demand relative to availability. Findings were presented to **SACRD** and **United Way of Bexar County**.

---

## Known Issues / Limitations

- Zip Code 78205 has been omitted from the analysis as it does not represent the actual demand for the area.
- Some interactions/shares lack user-entered ZIP codes (partially mitigated by `fill_zipcodes`).
- Data cleaning depends heavily on keyword lists (English/Spanish) → possible false positives/negatives.
- SACRD data has some inconsistencies that require careful filtering.
- Clustering is exploratory; not production-ready.
